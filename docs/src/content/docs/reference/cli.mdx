## Global flags

| Flag | Type | Description |
|------|------|-------------|
| `--verbose` | boolean | Enable verbose output |
| `--quiet` | boolean | Suppress non-essential output |

## Commands

### `run`

Execute an eval suite against the target function.

```bash
agent-eval-kit run [options]
```

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--suite` | string | (all) | Suite name to run |
| `--mode` | `live\|replay\|judge-only` | `live` | Execution mode |
| `--record` | boolean | `false` | Record fixtures during live mode |
| `--trials` | number | `1` | Number of trials per case |
| `--concurrency` | number | `1` | Max parallel case executions |
| `--timeout` | number | `30000` | Per-case timeout in milliseconds |
| `--run-id` | string | â€” | Previous run ID (for judge-only mode) |
| `--update-fixtures` | boolean | `false` | Force live mode + record (re-record all fixtures) |
| `--no-progress` | boolean | `false` | Disable live progress display |
| `--watch` | boolean | `false` | Re-run on file changes |
| `--strict-fixtures` | boolean | `false` | Fail on stale fixtures instead of warning |

### `record`

Shorthand for `run --mode=live --record`.

```bash
agent-eval-kit record --suite=smoke
```

### `compare`

Compare two eval runs side-by-side.

```bash
agent-eval-kit compare --base=<run-id> --compare=<run-id>
```

| Flag | Type | Description |
|------|------|-------------|
| `--base` | string | Base run ID (older, "before") |
| `--compare` | string | Compare run ID (newer, "after") |

### `list`

List recent eval runs.

```bash
agent-eval-kit list
```

### `cache`

Cache management commands.

#### `cache stats`

Show cache statistics for fixtures and judge cache.

```bash
agent-eval-kit cache stats
```

#### `cache clear`

Clear fixture or judge cache.

```bash
agent-eval-kit cache clear              # Clear fixture cache
agent-eval-kit cache clear --judge      # Clear judge cache only
agent-eval-kit cache clear --all        # Clear all caches
agent-eval-kit cache clear --suite=foo  # Clear fixtures for specific suite
```

### `doctor`

Run health checks on your eval setup.

```bash
agent-eval-kit doctor
```

### `init`

Initialize a new eval project with config and case files.

```bash
agent-eval-kit init
```

### `install-hooks`

Install git hooks for pre-push eval checks.

```bash
agent-eval-kit install-hooks
```

### `mcp`

Start the MCP server for AI assistant integration. Exposes 8 tools (list suites, describe config, list graders, validate config, run suites, list runs, compare runs, get run details) and 3 resources (config schema, case schema, grader reference).

```bash
agent-eval-kit mcp
```

Requires `@modelcontextprotocol/sdk` to be installed. See [MCP Server](/advanced/mcp-server/) for full tool reference and setup instructions.
